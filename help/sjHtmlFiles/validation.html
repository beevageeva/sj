Validation

There are 2 types of validation: the validation that is made in the page(when pressing next , prev or ok) for the fields that are only in that page and the final validation(made at final when pressing ok) for the fields that are not all in the same configuration page.

Validations made :
  • for the caches(including main memory)(third panel of configuration)
    • if the cache is enabled the number of entries must be between 2 and 2**32
    • the block size must be between 2 and 2**20
    • the number of sets must be between 1 and number of entries
    • the bus size must be between 8 and 128
    • the cache access time units must be between 0 and 50000
    • the block size of the caches must be less or equal to the page size of the main memory
      (In this case it is assumed that the caches have the same block size.)
  • for the memory configuration(first panel of configuration)
    • the maximum number of processes that can be loaded must be between 1 and 2**8
    • Total number number of virtual pages must be greater or equal to the number
      of real pages(the number of entries in the main memory)
    • Virtual address size must be less or equal to 32
  • for the page table(second panel of configuration)
    if is direct mapped
    • the number of levels must be between 1 and the number of virtual pages for each process
    • the sum of offset lengths must be equal to the number of virt pages for each process
    if is inverse mapped
    • the hash anchor size must be greater or equal than the number of entries in the main memory and
      less or equal than the total number of virtual pages
